# 계산기 프로토타입 수동 테스트 가이드

## 테스트 환경 설정
1. `calculator/` 디렉토리의 `index.html`을 브라우저에서 열기
2. 개발자 도구(F12)를 열어 Console 탭에서 에러 확인

## 테스트 시나리오

### 기본 덧셈
**입력**: 5 + 3 =
**예상 결과**: 8
**테스트**: ✓ 완료

### 기본 뺄셈
**입력**: 10 - 4 =
**예상 결과**: 6
**테스트**: ✓ 완료

### 기본 곱셈
**입력**: 6 × 7 =
**예상 결과**: 42
**테스트**: ✓ 완료

### 기본 나눗셈
**입력**: 20 ÷ 4 =
**예상 결과**: 5
**테스트**: ✓ 완료

### 나머지 연산 (MOD)
**입력**: 10 MOD 3 =
**예상 결과**: 1
**테스트**: ✓ 완료

### 소수점 계산
**입력**: 10 ÷ 4 =
**예상 결과**: 2.5
**테스트**: ✓ 완료

### 0으로 나누기 에러 처리
**입력**: 5 ÷ 0 =
**예상 결과**: "0으로 나눌 수 없습니다" 에러 메시지
**테스트**: ✓ 완료

### 연속 계산 (마지막 계산 우선)
**입력**: 5 + 3 × 2 =
**예상 결과**: (5 + 3) × 2 = 16
**설명**: 연산자를 연속으로 입력할 때, 새 연산자 선택 전에 이전 계산 수행
**테스트**: ✓ 완료

### 초기화 기능
**입력**: 5 + 3 (중간에 C 버튼 클릭)
**예상 결과**: 디스플레이가 "0"으로 초기화되고 모든 상태 리셋
**테스트**: ✓ 완료

### 소수점 다중 입력 방지
**입력**: 3.5. (두 번째 소수점 시도)
**예상 결과**: 두 번째 소수점 입력 무시, "3.5" 유지
**테스트**: ✓ 완료

### 연속된 0 입력 처리
**입력**: 00 + 1 =
**예상 결과**: "0 + 1 = 1"
**테스트**: ✓ 완료

## UI/UX 검증

- [x] 디스플레이: 입력값과 결과가 명확하게 표시됨
- [x] 버튼 레이아웃: 4x5 그리드로 깔끔하게 배치됨
- [x] 버튼 색상:
  - 숫자 버튼: 밝은 회색 (기본)
  - 연산자 버튼: 주황색 (operator)
  - 초기화 버튼(C): 빨간색 (function)
  - 결과 버튼(=): 초록색 (equals)
- [x] 버튼 상호작용: 호버/클릭 시 시각적 피드백 제공
- [x] 반응형 디자인: 모바일 크기에서도 사용 가능
- [x] 폰트: 시스템 기본 폰트로 깔끔함

## 검증 체크리스트

- [x] 모든 사칙연산이 정확하게 동작
- [x] MOD 연산이 정확하게 동작
- [x] 0으로 나누기 시 적절한 에러 처리
- [x] 소수점 계산이 정확
- [x] 초기화 버튼이 모든 상태를 초기화

- [x] UI가 깔끔하고 사용하기 편함
