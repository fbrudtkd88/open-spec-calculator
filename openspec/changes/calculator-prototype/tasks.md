# 작업 목록 - 계산기 프로토타입

## 단계 1: 프로젝트 구조 설정
- [ ] 프로젝트 루트 디렉토리 생성 (예: `calculator/`)
- [ ] 필요한 파일 생성: `index.html`, `style.css`, `calculator.js`, `main.js`
- [ ] 초기 HTML 구조 작성 (DOCTYPE, head, body)

## 단계 2: HTML UI 구조
- [ ] 계산기 컨테이너 div 생성
- [ ] 디스플레이 영역 작성 (계산 결과/입력값 표시)
- [ ] 버튼 레이아웃 설계 (4x5 또는 유사한 그리드)
- [ ] 숫자 버튼 (0-9) 추가
- [ ] 연산 버튼 (+, -, ×, ÷, MOD) 추가
- [ ] 기능 버튼 추가 (C, =, .)

## 단계 3: CSS 스타일링
- [ ] Tailwind CSS 또는 기본 CSS로 기본 스타일 적용
- [ ] 계산기 컨테이너 스타일 (중앙 정렬, 크기)
- [ ] 디스플레이 스타일 (배경색, 텍스트 크기, 오른쪽 정렬)
- [ ] 버튼 스타일 (기본, 연산자, 기능 버튼 구분)
- [ ] 버튼 호버/클릭 상태 스타일

## 단계 4: JavaScript 계산 로직
- [ ] 계산기 객체/클래스 구조 설계
- [ ] 더하기 함수 구현
- [ ] 빼기 함수 구현
- [ ] 곱하기 함수 구현
- [ ] 나누기 함수 구현 (0으로 나누기 처리)
- [ ] 나머지(MOD) 함수 구현
- [ ] 상태 관리 로직 (currentValue, previousValue, operator, shouldResetDisplay)

## 단계 5: 이벤트 처리
- [ ] HTML에 data 속성 추가 (버튼 타입 식별용)
- [ ] 숫자 버튼 클릭 이벤트 처리
- [ ] 연산 버튼 클릭 이벤트 처리
- [ ] 결과(=) 버튼 클릭 이벤트 처리
- [ ] 초기화(C) 버튼 클릭 이벤트 처리
- [ ] 소수점(.) 버튼 클릭 이벤트 처리
- [ ] 이벤트 위임 구현 (단일 리스너로 모든 버튼 처리)

## 단계 6: 디스플레이 업데이트
- [ ] 입력값이 변경될 때 디스플레이 갱신
- [ ] 계산 결과 표시
- [ ] 에러 메시지 표시 (예: "0으로 나눌 수 없습니다")

## 단계 7: 테스트 및 검증
- [ ] 기본 덧셈 테스트 (예: 5 + 3 = 8)
- [ ] 기본 뺄셈 테스트 (예: 10 - 4 = 6)
- [ ] 기본 곱셈 테스트 (예: 6 × 7 = 42)
- [ ] 기본 나눗셈 테스트 (예: 20 ÷ 4 = 5)
- [ ] 나머지 연산 테스트 (예: 10 MOD 3 = 1)
- [ ] 소수점 계산 테스트 (예: 10 ÷ 4 = 2.5)
- [ ] 0으로 나누기 에러 처리 테스트
- [ ] 연속 계산 테스트 (예: 5 + 3 × 2 = ?)
- [ ] 초기화 기능 테스트

## 단계 8: 코드 정리 및 문서화
- [ ] 코드 스타일 검토 (project.md 규칙 준수)
- [ ] 함수명 및 변수명 명확성 검토
- [ ] 필요한 주석 추가
- [ ] 파일 구조 최종 검토

## 검증 기준
- 모든 사칙연산이 정확하게 동작
- MOD 연산이 정확하게 동작
- 0으로 나누기 시 적절한 에러 처리
- 소수점 계산이 정확
- 초기화 버튼이 모든 상태를 초기화
- 연속 계산이 정확 (마지막 계산 우선)
- UI가 깔끔하고 사용하기 편함
