# 기본 연산 및 나머지 연산 명세

## 개요
계산기의 핵심 기능으로 사칙연산(+, -, ×, ÷)과 나머지 연산(MOD)을 지원합니다.

## ADDED Requirements

### Requirement: 덧셈 연산
계산기는 두 개의 숫자를 더할 수 있어야 합니다. SHALL support addition operation of two numbers.

#### Scenario: 기본 덧셈
- 사용자가 `5`, `+`, `3`, `=`을 순차적으로 입력
- 결과: `8`이 디스플레이에 표시됨
- 검증: Math.js 또는 수동 확인

#### Scenario: 소수점 덧셈
- 사용자가 `3.5`, `+`, `2.5`, `=`을 입력
- 결과: `6`이 디스플레이에 표시됨

#### Scenario: 음수 포함 덧셈
- 사용자가 `-5`, `+`, `3`, `=`을 입력
- 결과: `-2`가 디스플레이에 표시됨

### Requirement: 뺄셈 연산
계산기는 첫 번째 숫자에서 두 번째 숫자를 뺄 수 있어야 합니다. SHALL support subtraction operation.

#### Scenario: 기본 뺄셈
- 사용자가 `10`, `-`, `4`, `=`을 입력
- 결과: `6`이 디스플레이에 표시됨

#### Scenario: 음수 결과 뺄셈
- 사용자가 `5`, `-`, `10`, `=`을 입력
- 결과: `-5`가 디스플레이에 표시됨

### Requirement: 곱셈 연산
계산기는 두 개의 숫자를 곱할 수 있어야 합니다. SHALL support multiplication operation.

#### Scenario: 기본 곱셈
- 사용자가 `6`, `×`, `7`, `=`을 입력
- 결과: `42`가 디스플레이에 표시됨

#### Scenario: 소수점 곱셈
- 사용자가 `2.5`, `×`, `4`, `=`을 입력
- 결과: `10`이 디스플레이에 표시됨

### Requirement: 나눗셈 연산
계산기는 첫 번째 숫자를 두 번째 숫자로 나눌 수 있어야 합니다. SHALL support division operation.

#### Scenario: 기본 나눗셈
- 사용자가 `20`, `÷`, `4`, `=`을 입력
- 결과: `5`가 디스플레이에 표시됨

#### Scenario: 소수점 결과 나눗셈
- 사용자가 `10`, `÷`, `4`, `=`을 입력
- 결과: `2.5`가 디스플레이에 표시됨

#### Scenario: 0으로 나누기 에러
- 사용자가 `5`, `÷`, `0`, `=`을 입력
- 결과: 에러 메시지 (예: "0으로 나눌 수 없습니다") 표시
- 디스플레이는 초기화되거나 에러 상태로 유지됨

### Requirement: 나머지 연산(MOD)
계산기는 첫 번째 숫자를 두 번째 숫자로 나눈 나머지를 계산할 수 있어야 합니다. SHALL support modulo operation.

#### Scenario: 기본 MOD 연산
- 사용자가 `10`, `MOD`, `3`, `=`을 입력
- 결과: `1`이 디스플레이에 표시됨 (10을 3으로 나눈 나머지)

#### Scenario: MOD 0 입력
- 사용자가 `5`, `MOD`, `0`, `=`을 입력
- 결과: 에러 메시지 표시
- 이유: 0으로 나눗셈과 동일한 에러 처리

#### Scenario: 소수점 MOD 연산
- 사용자가 `7.5`, `MOD`, `2`, `=`을 입력
- 결과: `1.5`가 디스플레이에 표시됨 (또는 구현 방식에 따라)

### Requirement: 초기화 기능
계산기는 Clear(C) 버튼으로 모든 상태를 초기화할 수 있어야 합니다. SHALL support reset functionality.

#### Scenario: 계산 중 초기화
- 사용자가 `5`, `+`, `3`을 입력 후 C를 입력
- 결과: 디스플레이가 `0`으로 초기화됨
- 모든 내부 상태(피연산자, 연산자)가 초기화됨

#### Scenario: 결과 후 초기화
- 사용자가 `5`, `+`, `3`, `=`으로 `8`을 얻은 후 C를 입력
- 결과: 디스플레이가 `0`으로 초기화됨
- 다음 계산을 새로 시작할 수 있음

### Requirement: 소수점 입력
계산기는 소수점 숫자 입력을 지원해야 합니다. SHALL support decimal point input.

#### Scenario: 소수점 입력
- 사용자가 `3`, `.`, `5`를 입력
- 디스플레이에 `3.5`가 표시됨
- 다중 소수점 입력은 방지됨 (예: `3.5.2` 입력 불가)

### Requirement: 연속 계산
계산기는 연속 계산을 지원해야 합니다 (=를 누르지 않고 연산자 변경). SHALL support chained calculations.

#### Scenario: 연속 계산 (마지막 계산 우선)
- 사용자가 `5`, `+`, `3`, `×`, `2`, `=`을 입력
- 예상 결과: `(5 + 3) × 2 = 16` 또는 `5 + (3 × 2) = 11` (구현 방식에 따라)
- 주의: 프로토타입에서는 왼쪽에서 오른쪽으로 순차 계산으로 구현 예정

### Requirement: 디스플레이 표시
계산기 디스플레이는 현재 입력값 또는 계산 결과를 명확하게 표시해야 합니다. SHALL display current input or calculation result clearly.

#### Scenario: 입력 중 표시
- 사용자가 숫자를 입력 중일 때 입력값이 디스플레이에 표시됨
- 입력값은 오른쪽 정렬로 표시됨

#### Scenario: 계산 결과 표시
- 연산 후 결과가 디스플레이에 표시됨
- 다음 입력 시 자동으로 새로운 계산 시작

## 관련 기능
- 계산 이력 기능 (향후): 계산 이력에 저장할 기초 제공
- 테마/설정 기능 (향후): 디스플레이 색상, 글꼴 등 커스터마이징

## 참고사항
- 연산자 우선순위: 현재 프로토타입에서는 왼쪽에서 오른쪽으로 순차 계산
- 부동점 연산: JavaScript의 부동점 연산 특성 주의 (예: 0.1 + 0.2)
- 큰 숫자: 브라우저의 JavaScript 숫자 한계 고려
