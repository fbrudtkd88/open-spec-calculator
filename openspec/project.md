# Project Context

## 목적
사용자가 기본 사칙연산을 간편하게 수행할 수 있는 웹 기반 계산기 애플리케이션입니다. 깔끔하고 현대적인 인터페이스를 제공하며, 계산 이력 조회, 테마/설정 커스터마이징, 반응형 사용자 경험을 지원합니다.

## 기술 스택
- HTML5
- CSS3 (Tailwind CSS를 이용한 스타일링)
- Vanilla JavaScript (프레임워크 미사용)
- Jest 또는 Vitest를 이용한 단위 테스트

## 프로젝트 규칙

### 코드 스타일
- 변수와 함수명은 카멜케이스(camelCase) 사용
- HTML 클래스명과 CSS 선택자는 케밥케이스(kebab-case) 사용
- 의미 있는 변수명을 사용하여 가독성 높은 코드 작성
- 함수는 단일 책임 원칙을 따라 작은 단위로 구성
- 로직이 명확하지 않을 때만 주석 작성

### 아키텍처 패턴
- 관심사의 분리: HTML(구조), CSS(스타일링), JavaScript(로직)
- 기능별로 분리된 모듈 기반 조직 구조
- 계산기 버튼 클릭 이벤트는 이벤트 위임을 통해 처리
- DOM 조작에는 데이터 속성(data attributes) 활용

### 테스트 전략
- Jest 또는 Vitest를 사용하여 계산기 로직 단위 테스트 작성
- 모든 사칙연산(+, -, ×, ÷) 테스트
- 엣지 케이스 테스트: 0으로 나누기, 소수점 계산, 큰 숫자 처리
- 이력 조회 및 설정 기능 테스트
- 핵심 계산 로직에 대해 높은 커버리지 목표

### Git 워크플로우
- 커밋 메시지는 Conventional Commits 형식 따르기
- 새로운 기능 추가 시 피처 브랜치 생성
- 한 번에 하나의 기능 또는 버그 수정만 커밋
- 예시: `feat: 계산 이력 기능 추가` 또는 `fix: 0으로 나누기 처리`

## 도메인 컨텍스트
- 계산기는 덧셈, 뺄셈, 곱셈, 나눗셈의 기본 연산 지원
- 이력 기능으로 이전 계산 결과를 저장하고 사용자가 조회 가능
- 테마/설정: 다크모드 토글, 소수점 자릿수 설정
- 계산 디스플레이는 소수점 숫자와 큰 숫자 표현 지원

## 중요 제약사항
- 최신 브라우저(Chrome, Firefox, Safari, Edge) 호환
- 백엔드 불필요 - 모든 로직은 클라이언트 사이드에서 실행
- 가볍고 빠른 성능
- 접근성을 고려한 UI (WCAG 가이드라인 준수)

## 외부 의존성
- Vanilla JavaScript만 사용하므로 필수 의존성 없음
- 선택사항: Tailwind CSS CDN 또는 로컬 설정
